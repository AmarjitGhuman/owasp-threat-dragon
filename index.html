Threat Dragon is an open-source threat modelling tool from OWASP. To use it, you don't need to sign-up or sign-in and it is completely free. All your models are stored on your local device, so your data is never sent to anyone else or stored at some unknown location on the cloud.

It is currently in the early stages of development (an OWASP incubator project) so there might be some bugs.

If you make a model that you want to keep, you can print it to a file or hard copy. You can also save the raw model data as a file to share with friends or move between your devices or browsers.

Getting started

Starting from scratch

When you browse to Threat Dragon for the first time and hit https://threatdragon.azurewebsites.net you will see the Welcome page. To get started with your threat model click or tap

Create Local Model

and then

Create new threat model...

This takes you to the threat model edit page where you can enter general information about your model. The Title field is mandatory. All the rest are optional, but they provide context for your model. This can be useful if someone else has to pick the model up in the future. Although only the Title field is mandatory, your model will a be pretty useless without Diagrams. Add some diagrams to your model by clicking or tapping

Add a new diagram...

Name your diagram and then click or tap Add to confirm or Cancel if you change your mind

At this stage you are just listing the diagrams and naming them. You add all the diagram elements later.

Once you have entered all the details you need, click or tap Save. You can also Cancel to exit without saving, or Reload to undo any changes and revert to your last save.

Congratulations! You have got the basics done. Next step...mapping out your system in a diagram.

Starting from a file

If a friend or colleague has sent you a threat model file then you can import it into your local storage. Go to the threat models list at https://threatdragon.azurewebsites.net/#/threatmodels. To import the model, click or tap

Load a threat model from a file...

Now browse to your file. By default it will have file extension .dragon. Open it and you are done! Your new model should appear in your threat model list.

Loading a demo model

If you are wondering how to start, you can load a sample threat model. Go to the threat models list at https://threatdragon.azurewebsites.net/#/threatmodels. To download the sample, click or tap

Download a demo model...

This should give you some ideas on how to get started with your own model.

Threat model diagrams

Processes, data stores and actors

Add model elements to your diagram by clicking or tapping the relevant shape in the stencil on the left side of the diagram editor. Once added they can be selected by clicking them to see their properties and threats and dragged around the diagram. to delete an element, first select it and then click on the red icon it the elements top left corner...

Data flows and trust boundaries

Data flows and trust boundaries can be added to the diagram by clicking their shape in the stencil on the left side of the diagram editor. Once added, their ends can be dragged around the diagram. To connect the end of a data flow to a process, data store or actor, you can drag one of its ends onto the element.

An easier way to draw data flows between elements is to select your first element, then click the grey link tool, next to the red remove tool near the top right of the selected element. This turns the link tool green. Then, when you click another element, a new data flow will be created, linking the first element to the second.

Extra vertices can be added by clicking at some point on the line. These new vertices can also be dragged to position the data flow or trust boundary. Vertices can be removed by clicking the remove tool that appears when you mouse near to the vertex.

A data flow can be selected by clicking the Link options tool that appears when you mouse near to the link. Once selected you can edit its properties or add threats to it. Trust boundaries cannot be selected.

Data flows and trust boundaries can be deleted by clicking the red remove tool that appear when you mouse near to them.

Editing toolbar

The toolbar on the diagram editing page supports some general diagramming features:

Cancels the edit and returns to the threat model details view.

Zooms the diagram in and out.

Clears all the elements from the model.

Reloads the diagram from the last save, discarding any changes.

Generates threats for the selected element using the threat generation rule engine.

In the current alpha version of OWASP Threat Dragon, the threat generation rule engine is a stub, but you can still manually add threats to create a full threat model.

Saves the threat model to your local browser storage.

Element properties

To edit the properties of a model element, first select it. The element properties are shown on the right side of the diagram editor. In a future version of Threat Dragon, these properties will be used by the threat generation engine to suggest threats and mitigations for your model.

Adding and editing threats

To add threats to elements in your diagram, select an element and tap or click Edit Threats on the left side of the diagram editor. This will collapse the model element stencil and show the threats for the selected element. To add a new threat, tap or click

Enter the details for your threat in the threat dialog. The Title and STRIDE Threat Type are mandatory. When you are done, hit Save. Your new threat should appear. To edit it again, tap or click its title.

Sharing and exporting threat models

Sharing

If you would like to share a threat model between your devices or browsers or with a friend you can save the model data as a file. This can be done from the threat models list.

This will save the model data in a local file. The file name will be built from the threat model title by replacing spaces in the title with underscore characters (_). So for example, a threat model called My Cool Model will result in a file called My_Cool_Model.dragon. Another use for this file is if you would like to keep the model in your source control system.

Printing

You can create a printed version of the model (or a PDF or XPS version if you have the appropriate printer driver). This can be done from either the threat models list or the threat model detail page.

Print

This will create a new browser tab with a report style view of the threat model showing the summary data, all the diagrams and all the elements from each diagram with their properties and threats. This can then be printed.

Threat generation rules

Threat Dragon is still working towards its first milestone - an alpha release. At this point, the threat generation rules so far are just a stub to prove the technical approach. A more fully functional rule set is planned for the beta release (milestone 2), as described in the roadmap. Threat Dragon's alpha release aims to provide a basic threat modelling experience, but for the time being, you are on your own in terms of identifying threats and mitigations.
